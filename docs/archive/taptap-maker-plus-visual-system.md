## 1. 文档目的

这份文档是 TapTap Maker Plus / 制造++ 的第二份规划文档，专门约束视觉方向、界面结构、交互逻辑、动效与后续开发任务。

本项目不是重新做一个 Maker API，也不是替代 TapTap Maker MCP。制造++要做的是本地桌面级工作台：第一阶段先实现本地 Web 内核，由 Fastify 内置 MCP client 直接连接 TapTap Maker MCP，把 MCP 已经提供的图片、视频、音乐、3D、构建、项目状态、文件落盘能力做成更清晰、更可控、更适合长期维护的可视化工作台。

本文档解决三个问题：

- 后续 UI 不能再像临时调试面板，要像 TapTap 官方产品体系里的本地创作后台。
- 后续开发任务修改 UI 时，必须知道每个面板来自哪类 TapTap 页面、为什么这样摆、交互应该怎么做。
- 亮色和暗色主题必须作为同一套系统实现，不能出现局部亮、局部暗、局部硬编码的割裂状态。

命名统一：

- 英文名：`TapTap Maker Plus`
- 中文名：`制造++`
- 历史拼写错误已统一清理，后续只使用 `TapTap`。

## 3. 总体设计结论

TapTap 的官方视觉不是“花哨大屏”，而是高密度但克制的产品系统。核心语言如下：

- 亮色优先：主站状态明确出现 `themeMode: light`，制造++第一阶段以亮色为主视觉目标。
- 品牌色收敛：TapTap 标志使用 `#00D9C5`，AI 入口强调色出现 `#00CDBA`。制造++只能把绿色作为高价值动作、选中态、AI 状态、生成任务进度的核心强调色，不能把整页染成单一绿色。
- 信息密度高：开发者后台使用左导航、顶部身份胶囊、中间内容表格、右侧 Agent 面板，是制造++桌面端主结构的直接参考。
- 参考优先级明确：`参考.html` 中的 TapTap 制造应用壳适合生成、物料确认、素材选择流程，不适合直接作为制造++桌面外壳；桌面外壳必须采用开发者后台式工作台。
- 表格和列表是主干：游戏管理页以 `游戏信息`、`上次更新`、`近30天数据`、`评分` 建立操作台语义。制造++资产、任务、项目也应有列表/表格视图，不只做卡片墙。
- 媒体要真实可见：官网内容里有 `_tap_screenshot`、`_tap_appicon`、`_tap_banner` 等媒体资源形态。制造++的图片、视频、模型、音乐都必须有真实缩略图、预览或波形占位，不能只显示文件名。
- 右侧面板是工作台的一部分：开发者后台把 Agent 面板作为右侧可调整、可最小化、可切换布局的生产组件。制造++第一阶段右侧以 Inspector / MCP 状态 / 任务详情为主，后期再接 Agent 编排。
- 素材选择和生成是流程组件：TapTap 制造页面里有选择截图、AI 自动生成物料、步骤条、确认按钮。制造++的生成工具应该用流程化表单和节点流表达输入、参考、执行、输出、确认，但不承诺发布到 TapTap 或上传到官方平台。

## 4. 页面逐项视觉拆解

### 4.1 TapTap 制造应用壳

证据点：

- 页面标题为 `TapTap 制造`。
- SEO 描述为 `TapTap 制造 是一款专为游戏爱好者打造的 零门槛 AI 游戏创作智能体。想到就能做到！`。
- 根布局出现 `mobile-full-height`、`bg-background`、`dark:bg-gray-900`、`safe-area-top`。
- 顶部项目名区域有 `活下去`、编辑名称按钮、`历史会话`。
- 底部 tab 有 `对话`、`预览`、`发布`。

视觉结构：

- 这是偏移动端/窄屏的生成应用壳。
- 顶部是项目身份和核心动作；底部是对话、预览、发布三段式导航。
- 中间内容按“当前状态/历史会话/发布配置”滚动呈现。

组件语言：

- 图标按钮尺寸集中在 `h-8 w-8`。
- 发布按钮使用 `text-primary bg-primary/10 rounded-full`，hover 后切到 `bg-primary text-white`。
- 历史会话卡片使用 `rounded-lg border p-3`，hover 用 `can-hover:bg-accent`。
- hover 后显示更多操作，证据里有 `md:opacity-0 md:group-hover:opacity-100`。

交互逻辑：

- 项目名可编辑，但编辑入口默认弱化，hover 时增强。
- 历史会话是可点击列表项，右侧更多操作 hover 才出现。
- 底部 tab 在窄屏负责主模块切换。

制造++迁移点：

- 多项目顶部区需要保留“当前项目名 + 项目切换 + 状态”的对象感。
- 历史生成任务、节点流记录、对话记录应使用同类列表项，不要只放大卡片。
- 窄屏模式保留底部 `项目 / 工作台 / 资产 / 智能编排` 类型 tab，但桌面端不使用底部主导航。

不照搬点：

- 制造++是本地桌面管理台，不能以移动端单列作为主结构。
- `dark:bg-gray-900` 这种局部暗色类不能直接混进组件；主题必须由根 token 驱动。

### 4.2 TapTap 制造物料确认流

证据点：

- 发布区域出现 `发布到 TapTap`、`发布厂商`、`扫码在手机上测试`、`发布物料`。
- 上传卡出现 `点击上传游戏截图`、`截图将用于生成发布物料`。
- AI 物料说明出现 `AI 将根据截图自动生成`。
- 物料类型包括 `游戏名称`、`游戏图标`、`游戏截图`、`宣传 Banner`、`横版封面`、`竖版封面`。
- 步骤条出现 `生成物料`、`发布`。
- 主要确认按钮为 `确认发布`。
- 动效类出现 `animate-in fade-in slide-in-from-left-4 duration-300`。

视觉结构：

- 参考页里的发布流不是单个表单，而是“厂商选择 - 测试版本 - 物料生成 - 发布确认”的纵向流程。制造++不实现官方发布，只吸收这种“身份确认 - 素材选择 - 生成结果 - 人工确认 - 写入项目”的流程结构。
- 参考页的物料上传卡大面积使用虚线边框、圆角和浅绿色背景。制造++应转译为“素材导入卡”，让用户明确知道这是可拖拽/可选择项目内素材的区域。
- AI 生成物料区域用浅灰底 `bg-gray-50 rounded-2xl p-4` 容纳结果说明。

组件语言：

- 素材导入卡：参考 `border-2`、`border-dashed-sparse`、`rounded-2xl`、`hover:border-primary`、`hover:bg-primary/5`。
- 素材导入中心图标：白底圆形、`shadow-sm`、大号 plus 图标、`text-primary`。
- 步骤圆点：当前步骤 `bg-primary text-white`，未完成步骤 `bg-gray-200 text-gray-400`。
- 确认按钮：全宽、`h-12`、`rounded-full`、`text-lg`。

交互逻辑：

- 未满足条件时 `生成物料` 和 `确认发布` 禁用。
- 素材进入后，AI 自动生成说明区域以 300ms 左侧滑入 + 淡入呈现。
- 测试二维码固定指向测试版本，更新测试版本是独立次级操作。

制造++迁移点：

- 图片生成、视频生成、模型生成都应该有“输入区 - 参数区 - 执行区 - 输出区 - 写入项目”的流程感。
- 节点流里每个节点也应该有“输入资产 / 生成工具 / 输出资产 / 项目落盘”四段信息。
- 生成结果不能只进入聊天记录，必须进入资产库并可追踪到来源节点。
- `确认发布` 在制造++里只能转译为 `确认写入项目`、`确认保存资产`、`确认加入工作流`，不得做成 TapTap 官方发布动作。

不照搬点：

- 发布页的移动优先布局不适合桌面多面板。制造++只吸收“流程组件”和“AI 物料卡”的视觉语言。

### 4.3 TapTap 官网主站

证据点：

- 根布局包含 `main-layout flex flex-col main-layout--window-scroll main-layout--sidebar-ready main-layout--with-tabbar`。
- 顶栏包含 `main-layout-header main-layout-header--default`。
- 移动端 open app 条包含 `下载 App`。
- 顶部品牌 SVG 使用 `fill="#00D9C5"`。
- 搜索框为 `tap-search-box`。
- 搜索占位轮播文本包含 `TapTap 制造`、`尘白禁区 充值退还`、`绝尘漂移`、`地牢猎手6`、`诡秘之主`。
- AI 入口为 `tap-ai-entry`，图标里出现 `#52565A` 与 `#00CDBA`。
- 顶部右侧按钮包含 `下载手机 APP`、`下载 PC 版`。
- 底部状态中出现 `themeMode: light`。
- 媒体资源大量使用 `_tap_screenshot`、`_tap_appicon`、`_tap_banner`。

视觉结构：

- 官网顶栏是“品牌标识 - 中心搜索 - 右侧动作”的三段式。
- 搜索是中心对象，不是简单输入框；它包含搜索图标、占位轮播、AI 入口和结果动作区域。
- 主站的媒体内容以真实图像为视觉中心，图片 URL 后缀把 app icon、screenshot、banner 分为不同用途。

组件语言：

- 品牌色是极亮青绿，文本用深蓝或深灰保持克制。
- 顶栏按钮采用低装饰、高可读的文字按钮。
- 搜索占位文本有当前/前一项状态，适合做纵向滚动或淡入淡出。

交互逻辑：

- 搜索框可成为命令入口。
- AI 入口作为搜索旁边的独立动作，不与普通搜索混在一起。
- 下载按钮按平台分组，说明同一类动作应并列但不互相抢视觉。

制造++迁移点：

- 顶栏中心做全局命令搜索：搜索项目、工具、资产、节点、历史任务。
- AI 入口在第一阶段只做命令搜索旁的后期扩展位；后续可接入智能编排，但不能替代 MCP client 直接调用链路。
- 资产库必须区分图标、截图、Banner、视频、音频、模型，不再只按扩展名堆文件。

不照搬点：

- 官网是面向消费者的内容入口，制造++是生产工具。制造++不能做大面积 feed，也不能把下载按钮式营销动作搬进工作台。

### 4.4 TapTap 开发者后台 / 游戏管理

证据点：

- 根为 `ant-app css-var`。
- 主布局变量包含 `--agent-panel-width: 360px`、`--agent-docked-panel-width: 480px`、`--agent-docked-expanded-min-width: 480px`、`--agent-docked-expanded-duration: 360ms`、`--agent-floating-width: 420px`。
- 左侧导航 `aria-label="工作区菜单"`。
- 导航组有 `游戏`、`厂商管理`、`官方号与制作人员`、`财务`、`客服`。
- 当前选中项为 `游戏管理`。
- 旧版提示为 `新版功能持续优化中，遇到问题或缺失功能点击可回旧版`。
- 用户区显示 `黎塔`、`已认证`、`ID:399463`。
- 内容页标题为 `游戏管理`。
- 通知 tabs 为 `平台公告`、`最新通知`。
- 公告项包含 `开发者违规处罚公告`。
- 列表工具条包含卡片视图、列表视图、`全部状态`、`共 1 款游戏`、`搜索游戏名称`。
- 表格列为 `游戏信息`、`上次更新`、`近30天数据`、`评分`。
- 表格行包含 `幻海航迹`、`创意工坊 · V-20260530`、`未发布`、`浏览0`、`下载0`、`预约0`。

视觉结构：

- 左侧是可折叠导航栏。
- 顶部是品牌/厂商/面包屑/外链/账号动作。
- 中央是公告支持区 + 列表模块。
- 列表模块有明确工具条，支持视图切换、状态筛选、搜索和总量说明。
- 表格行把媒体封面、名称、平台、状态、指标集中在一行，信息密度高但可扫描。

组件语言：

- 左导航分组标题弱化，选中项强化。
- 顶部面包屑使用胶囊结构，适合承载当前厂商、项目或环境。
- 状态使用 `statusPill`，内部有圆点和文本。
- 表格封面使用真实图像，并有 blur fallback 层。
- 工具条使用 radiogroup 做卡片/列表切换，符合专业后台习惯。

交互逻辑：

- 左导航可折叠。
- 视图切换是 radio 行为，不是普通按钮。
- 状态筛选是 combobox。
- 表格行可点击，更多操作用独立 icon button。
- 公告区以 tab 切换类型，紧凑展示。

制造++迁移点：

- 制造++桌面端主结构直接采用：左项目/模块导航 + 中央工作区 + 右 Inspector / 后期智能编排面板。
- 资产库、任务中心、项目中心都必须提供列表视图和卡片视图。
- 多项目管理应把项目状态做成类似游戏管理表格：项目名、绑定状态、MCP 状态、最近生成、最近构建、当前分支、远端同步。

不照搬点：

- 开发者后台的财务、客服、官方号导航不适合制造++；制造++应改成项目、资产、生成、节点流、构建、设置。

### 4.5 TapTap 开发者后台 Agent 面板参考

证据点：

- 右侧面板为 `_agentColumn`、`_agentColumnCard`。
- 面板有 `data-component="AgentPanel"`。
- 有 `_agentLoadingGlowLayer` 和四条边缘 glow。
- 有 `aria-label="调整 Agent 面板宽度"`。
- toolbar 显示 `新对话`。
- 工具按钮包含 `新建对话`、`切换为侧边栏`、`删除当前对话`、`最小化`。
- 空状态标题为 `游戏状态速览`。
- 空状态文本为 `当前管理 1 款游戏，1 项状态提醒，暂无紧急事项。`
- 强调文本为 `1 款游戏暂无有效评分`。
- 快捷动作包含 `分析全部游戏数据表现`、`分析审核状态`、`找最近需要继续处理的游戏`。
- composer placeholder 为 `输入消息,Enter 发送,Shift+Enter 换行`。
- 上传能力包含 `上传图片`、`上传视频`、`长按说话`、`发送`。

视觉结构：

- Agent 面板是固定在右侧的生产侧栏，宽度可调整。
- 面板有独立 toolbar、消息体、快捷动作区、底部 composer。
- 空状态不是空白，而是基于当前业务对象给出摘要和推荐动作。

组件语言：

- 面板边缘 glow 表示 Agent 正在活动或可用，但默认不喧宾夺主。
- 快捷动作是 fit-content 的次级按钮，便于多行排列。
- composer 高度从 `44px` 起，最大高度 `132px`，适合短指令和多行 prompt。

交互逻辑：

- 支持新对话、删除、最小化、切换侧边栏。
- 支持图片、视频、音频输入，但上传视频在证据中为 disabled。
- Enter 发送，Shift+Enter 换行。

制造++迁移点：

- 第一阶段右侧面板必须知道当前选中的项目、资产、节点和任务，并显示 Inspector、MCP 状态、任务详情。
- 快捷动作要动态来自当前面板，例如资产库显示“分析这批图片风格”“生成同风格图标”“把结果写入当前项目”。这些快捷动作第一阶段应调用本地 MCP client API。
- Agent 只作为后期智能编排层，不是第一阶段主界面替代品；参数面板、节点流、任务队列仍由用户可视化控制。

不照搬点：

- 不能只做聊天面板。后期智能编排能力必须复用任务状态、节点流、资产元数据和项目文件上下文。

## 5. 制造++融合后的主界面目标

制造++桌面端采用“五区结构”：

```text
--------------------------------------------------------------------------------+
| Top Bar: TapTap Maker Plus / 制造++ | Project Switch | Command Search | Status |
+------------+-----------------------------------------------+-------------------+
| Project    | Workbench                                     | Inspector / Later AI |
| Tree + Nav | - Dashboard                                   | - Context Summary |
|            | - Asset Hub                                   | - Quick Actions   |
|            | - Generation Studio                           | - Chat Composer   |
|            | - Workflow Canvas                             | - Node Inspector  |
|            | - Build Center                                |                   |
+------------+-----------------------------------------------+-------------------+
| Bottom Task Queue: generating, uploading, building, syncing, errors            |
+--------------------------------------------------------------------------------+
```

### 5.1 顶栏

来源融合：

- TapTap 官网的三段式顶栏。
- 开发者后台的厂商/项目胶囊。
- TapTap 制造的项目名与 primary action 按钮。

制造++要求：

- 左侧显示 `TapTap Maker Plus` 与 `制造++`，不能只显示小 logo。
- 中间显示当前项目胶囊：项目名、路径摘要、MCP 连接状态、远端环境。
- 中心偏右放命令搜索，搜索项目、工具、资产、节点、任务。
- 右侧放明确动作：同步状态、构建、打开项目目录、主题切换、设置。

### 5.2 左侧项目树与导航

来源融合：

- 开发者后台的 `工作区菜单`。
- 用户需求中的多项目管理与项目文件管理。

制造++要求：

- 左栏上半部是本地 Maker 项目列表。
- 左栏下半部是当前项目模块导航。
- 模块分组：
  - `项目`
  - `资产`
  - `生成`
  - `节点流`
  - `构建`
  - `设置`
- 项目列表必须显示状态点：已绑定、MCP 可用、远端同步异常、构建中。
- 支持折叠，但折叠后仍显示项目状态点和 tooltip。

### 5.3 中央工作区

来源融合：

- 开发者后台列表/表格。
- TapTap 制造生成/物料确认流程。
- TapTap 官网媒体缩略图体系。

制造++要求：

- 默认页不是营销首页，而是当前项目工作台。
- 工作台第一屏必须看得懂当前在管哪个项目、有哪些资产、有哪些任务、哪些工具可用。
- 每个模块都要有明确空状态，不允许出现“空白大卡片”。

核心模块：

- 项目总览：项目路径、MCP 状态、最近任务、最近构建、资产统计。
- 资产库：图片、视频、音频、模型、节点流输出。
- 生成工作室：图片、视频、音乐、3D 分面板。
- 节点流：参考图、参考视频、提示词、生成工具、人工确认、写入项目。
- 构建中心：本地变更、提交状态、远端构建状态、runtime log。

### 5.4 右侧 Inspector / 后期智能编排扩展位

来源融合：

- 开发者后台 `AgentPanel`。
- 节点流工具的属性面板需求。

制造++要求：

- 右侧第一阶段默认是 Inspector / MCP 状态 / 任务详情面板。
- 当选中节点、资产、任务时，右侧切换为对应属性 Inspector。
- 智能编排 tab 保留为后期扩展位，第一阶段不依赖 Agent 执行 MCP 调用。
- 宽度基线参考证据：浮动宽度 `420px`，dock 宽度 `480px`，普通宽度 `360px`。
- 可调整宽度、可最小化、可切换侧栏模式。

### 5.5 底部任务队列

来源融合：

- TapTap 制造的步骤条。
- 生成任务长耗时场景。

制造++要求：

- 所有 MCP 调用进入统一任务队列。
- 任务项显示：工具名、项目、输入摘要、状态、进度、输出路径、错误详情。
- 视频、音乐、3D 这类长任务必须支持后台继续运行和重新查询。
- 失败状态必须保留 raw result，供开发者和后期智能编排层复盘。

### 5.6 桌面端操作模型硬规则

制造++桌面端必须是常驻分栏工作台，不允许退化成一页式长滚动页面。视觉对齐 TapTap 只是基础，操作模型也必须对齐开发者后台式的分栏生产工具。

硬规则：

- 顶栏常驻：项目切换、命令搜索、MCP 状态、构建入口始终可见。
- 左栏常驻：项目树和模块导航始终可见；可折叠为图标栏，但不能因为中央内容滚动而消失。
- 右栏常驻：Inspector / 后期智能编排扩展位始终存在；可最小化、可切换、可调整宽度，但不能被页面滚动带走。
- 底部任务队列常驻：生成、素材导入、写入项目、构建、同步任务跨模块持续可见；可折叠，但不能藏进中央页面底部。
- 只有中央工作区滚动：资产列表、任务列表、生成表单、构建日志在中央区域内部滚动。
- 导航不重置上下文：切换 `项目 / 资产 / 生成 / 节点流 / 构建 / 设置` 时，只替换中央工作区，当前项目、右侧上下文、任务队列不丢失。
- 节点流画布内部 pan/zoom，不改变外层 shell；画布工具栏固定在中央工作区顶部或侧边。
- 资产库网格和表格只在中央工作区内部滚动，不推动顶栏、左栏、右栏、底栏。
- 后期智能编排 composer 固定在右侧面板底部；第一阶段右侧详情内容在面板内部滚动。
- 任务队列固定在底部 shell 区域，展开后显示详情，折叠后保留任务状态点和错误提示。
- 生成/物料确认流程只能作为中央工作区流程面板、右侧 Inspector、或覆盖在中央工作区内的 modal/drawer 出现，不能整页替换五区工作台。
- `发布到 TapTap`、`发布物料`、`生成物料`、`确认发布` 这类词只作为 TapTap 制造视觉证据；制造++中必须转译为 `写入项目`、`保存资产`、`加入工作流`、`构建` 等本地可执行动作。
- 窄屏才允许移动端 tab 化；桌面端禁止使用移动端单列逻辑。

## 7. 亮色 / 暗色主题策略

### 7.1 主题目标

第一阶段目标：

- 亮色主题作为默认主题。
- 暗色主题完整可用，但不作为第一眼主视觉。
- 主题切换不改变布局密度、圆角、动效，只改变语义 token。

亮色主题气质：

- 背景轻灰，面板白色，边框低对比。
- TapTap 绿用于品牌、选中、主要动作、AI 状态。
- 状态色少量使用蓝、黄、红、紫，但只用于状态和分类，不做大面积背景。

暗色主题气质：

- 深灰而不是纯黑。
- 面板层级靠明度差和边框，不靠大面积发光。
- TapTap 绿在暗色下仍然保持品牌识别，但降低填充面积，避免霓虹感。

### 7.2 防止半亮半暗的规则

硬规则：

- 全局只允许一个根主题入口：`<html data-theme="light">` 或 `<html data-theme="dark">`。
- 组件不得自行判断系统主题，不得在局部切 `dark` class。
- 组件不得直接使用 `#fff`、`#000`、`bg-white`、`dark:bg-gray-900` 做业务 surface。
- 第三方组件必须通过主题桥接层映射到制造++ token。
- toast、popover、modal、context menu、tooltip、command palette 必须跟随根主题。
- 媒体缩略图外框使用中性 `--surface-muted`，不能让白色截图在暗色主题里无边界漂浮。
- 主题切换时只过渡 color/background/border/shadow，禁止过渡 layout 尺寸。

验收截图：

- `1440x900` 亮色。
- `1440x900` 暗色。
- `390x844` 亮色。
- `390x844` 暗色。

每张图检查：

- 顶栏、左栏、中间、右栏、底部任务队列主题一致。
- popover、toast、右键菜单主题一致。
- 资产缩略图边框可见。
- 所有文字在按钮和卡片内不溢出。

## 8. 交互逻辑与动效系统

### 8.1 导航与面板

左栏折叠：

- 点击折叠按钮后，图标保留，文本隐藏。
- 动效时长使用 180ms 到 220ms。
- 折叠状态下 hover 显示 tooltip。

右侧 Inspector / 后期智能编排面板：

- 初始宽度可使用 `360px`。
- dock 宽度可使用 `480px`。
- 浮动宽度可使用 `420px`。
- 展开/收起/布局切换使用 `360ms`。
- resize handle 必须有可访问标签，参考 `调整 Agent 面板宽度`。

中央工作区：

- 模块切换使用淡入和轻微位移，时长 180ms 到 240ms。
- 大流程面板进入可以使用 300ms，参考 `duration-300`。
- 表格排序、筛选、搜索不得触发布局大跳动。

### 8.2 生成与素材导入

素材选择区：

- hover：边框切品牌色，背景使用品牌色 5% 透明度。
- drag over：边框加粗或使用更高透明度背景，但不改变卡片尺寸。
- 文件进入后生成缩略图队列，不直接跳转页面。

生成任务：

- 创建任务后进入底部任务队列。
- 当前面板内显示局部 loading。
- 右侧 Inspector 显示任务摘要和下一步建议；后期智能编排面板可复用这些上下文。
- 成功后输出资产进入资产库，并标注来源工具和来源节点。

### 8.3 节点流

节点流是制造++区别于普通工具页的关键界面。

节点类型：

- `Reference Image`
- `Reference Video`
- `Prompt`
- `Image Generation`
- `Video Generation`
- `Music Generation`
- `3D Model Generation`
- `Manual Review`
- `Write To Project`
- `Build`

节点视觉：

- 节点主体使用 `--surface-panel`。
- 节点选中使用品牌色边框，不使用大面积填充。
- 节点端口使用小圆点，输入在左、输出在右。
- 长耗时节点顶部显示细进度条。
- 失败节点用红色状态点和错误摘要，不改变节点整体为红色大块。

连线逻辑：

- 参考类节点输出只能接到支持引用输入的生成节点。
- 生成节点输出接资产节点或人工确认节点。
- 写入项目节点必须显示目标项目与目标路径。
- 构建节点必须显示当前项目、分支和提交状态。

### 8.4 Toast 与系统反馈

证据中的 Sonner 样式显示 toast transform/opacity/height 使用 `.4s`，box-shadow 使用 `.2s`。制造++可采用同类节奏：

- 普通成功/失败 toast：进入 240ms，退出 180ms。
- 长任务 toast 不常驻，转入底部任务队列。
- 错误 toast 必须提供“查看详情”，详情打开任务记录。
- 支持 `prefers-reduced-motion`，关闭非必要动效。

## 9. 制造++核心面板规划

### 9.1 项目中心

目标：

- 管理本地多个 TapTap Maker 项目。
- 明确当前绑定项目、MCP runtime、远端环境、同步状态。

布局：

- 左：项目列表。
- 中：项目表格。
- 右：项目详情、MCP 状态和任务建议。

表格列：

- 项目名
- 路径
- 绑定状态
- MCP 状态
- 最近资产
- 最近构建
- Git 状态
- 操作

视觉重点：

- 状态点 + 状态胶囊。
- 项目封面或默认图标。
- 表格和卡片视图切换。

### 9.2 资产库

目标：

- 管理当前项目内由 MCP 生成或用户导入的图片、视频、音乐、模型。

布局：

- 顶部工具条：类型筛选、来源筛选、搜索、排序、视图切换。
- 主区域：卡片网格 / 列表表格。
- 右侧：选中资产详情，包含预览、路径、来源、可用操作。

视觉重点：

- 图片：真实缩略图。
- 视频：首帧 + 时长角标。
- 音频：波形或频谱占位 + 时长。
- 模型：渲染预览或模型图标 + 文件格式。
- 所有资产显示来源节点或来源工具。

### 9.3 生成工作室

目标：

- 将图片、视频、音乐、3D 生成工具做成可控面板，而不是每次手写 prompt。

布局：

- 左：工具类型 tabs。
- 中：参数表单和参考素材槽。
- 右：实时预览、历史输出、参数摘要和 MCP 状态。
- 底部：执行按钮和任务状态。

视觉重点：

- 参考素材槽使用 TapTap 制造素材选择卡语言。
- 主要动作使用品牌色圆角按钮。
- 高级参数折叠，不在首屏堆满。
- 生成流程和物料确认流程使用同一套流程组件：输入素材、参数、MCP 生成、人工确认、写入当前项目。
- TapTap 制造里的 `发布厂商`、`扫码在手机上测试`、`发布物料`、`生成物料`、`确认发布` 应转换为制造++里的项目身份、测试版本、生成输出、任务执行、写入项目确认，不照搬成独立移动页。
- 执行中的生成、素材导入、写入项目任务必须同步到底部任务队列，不能只停留在流程面板内部。

### 9.4 节点流画布

目标：

- 类似即梦一类参考节点/生成节点思路，但落到本地 MCP 和项目资产管理。

布局：

- 左：节点组件库。
- 中：画布。
- 右：节点属性 / 后期智能编排扩展位。
- 底：运行日志和任务队列。

视觉重点：

- 节点卡片克制，不做赛博风。
- 连接线清晰，端口语义明确。
- 节点运行状态和任务队列同步。

### 9.5 构建中心

目标：

- 管理 Maker 构建、提交、远端运行状态、runtime log。

布局：

- 顶部：当前项目构建状态。
- 中：构建历史列表。
- 右：日志详情。
- 底部：本地变更摘要。

视觉重点：

- 日志使用等宽字体，但周围仍然是 TapTap 风格面板。
- 失败信息结构化显示：阶段、命令、错误、建议动作。
- 构建按钮必须高可见，但只有状态满足时可点击。

### 9.6 Inspector / 后期智能编排扩展面板

目标：

- 第一阶段让 Inspector 与当前项目、资产、节点流、任务状态联动。
- 后期让智能编排层复用同一套上下文。

布局：

- 顶部：面板模式、布局切换、最小化。
- 中：当前对象速览、MCP 调用状态、任务记录、快捷动作。
- 底部：后期智能编排 composer 扩展位。

视觉重点：

- 空状态必须根据当前项目和 MCP client 状态生成。
- 快捷动作不写长段说明，使用短按钮。
- composer 支持图片/视频/音频附件，但禁用态要明确。

## 10. 开发任务拆分

后续拆分开发任务时，按下列工作单分配。每个任务必须读本文档和实际代码，不允许只凭截图印象改 UI。

### 10.1 Visual Token Task

任务：

- 从当前前端代码中找出所有硬编码颜色、圆角、阴影、spacing。
- 建立 `theme.css` 或同等 token 层。
- 将组件样式改为语义 token。

输出：

- token 文件。
- 被替换的硬编码列表。
- 亮色/暗色截图。

验收：

- 不再出现局部 `bg-white` 与 `dark:bg-gray-900` 混用导致的割裂主题。

### 10.2 Layout Task

任务：

- 实现五区结构：顶栏、左项目树、中工作区、右 Inspector / 后期智能编排扩展位、底部任务队列。
- 桌面端优先，兼容窄屏。
- 保留面板 resize 与折叠状态。

输出：

- 主 layout 组件。
- 响应式断点说明。
- 桌面/移动截图。

验收：

- 第一屏能看出这是制造++的本地 Maker 管理工具，不是普通 chat dashboard。

### 10.3 Project Tree Task

任务：

- 设计多项目列表、当前项目胶囊、项目状态点。
- 把项目切换、当前项目路径、MCP 状态放进 UI。

输出：

- 项目树组件。
- 项目状态组件。
- 空状态和错误状态。

验收：

- 用户可以一眼知道当前选中哪个 TapTap Maker 项目。

### 10.4 Asset Hub Task

任务：

- 设计资产库网格和列表双视图。
- 图片、视频、音频、模型采用不同缩略图策略。
- 资产详情显示来源工具、来源节点、落盘路径。

输出：

- Asset grid。
- Asset table。
- Asset inspector。

验收：

- 不打开文件系统也能看懂当前项目有哪些生成结果。

### 10.5 Generation Studio Task

任务：

- 将 `generate_image`、`batch_generate_images`、`edit_image`、`create_video_task`、`text_to_music`、`create_3d_model_task` 做成参数面板。
- 参考素材槽使用素材导入卡语言。
- 所有生成任务进入底部任务队列。

输出：

- 图片生成面板。
- 视频生成面板。
- 音乐生成面板。
- 3D 生成面板。

验收：

- 用户不需要手写完整工具 JSON，也能准确控制关键参数。

### 10.6 Workflow Canvas Task

任务：

- 调研并实现节点流基础结构。
- 节点类型覆盖参考素材、prompt、生成工具、人工确认、写入项目、构建。
- 节点执行与任务队列绑定。

输出：

- 节点流画布。
- 节点库。
- 节点属性面板。

验收：

- 可以把参考图连接到图片/视频生成节点，把输出写入项目资产库。

### 10.7 Motion Task

任务：

- 建立统一动效时长、easing、进入/退出规则。
- 实现面板折叠、节点选中、素材导入 hover、任务状态、toast 动效。
- 支持 `prefers-reduced-motion`。

输出：

- motion token。
- 动效示例页。
- reduced motion 检查结果。

验收：

- 动效增强信息理解，不造成跳动或干扰。

### 10.8 Accessibility Task

任务：

- 检查导航、按钮、tab、radiogroup、combobox、dialog、tooltip、resize handle。
- 建立键盘可用性标准。
- 重点检查后期智能编排 composer、节点流和素材导入区域。

输出：

- aria 规范清单。
- keyboard flow。
- 问题修复列表。

验收：

- 关键操作不用鼠标也能完成。

### 10.9 Verification Task

任务：

- 用 Playwright 检查桌面和移动端截图。
- 检查亮色/暗色一致性。
- 检查文字溢出、重叠、空白、canvas 非空、缩略图加载。

输出：

- 截图报告。
- 失败项列表。
- 修复建议。

验收：

- 每个核心页面在 `1440x900`、`390x844` 下无明显重叠和主题割裂。

## 11. 实施顺序

第一阶段：视觉骨架

- 建立主题 token。
- 重做主 layout。
- 做项目树、顶栏、右侧 Inspector 壳和后期智能编排扩展位。
- 做底部任务队列静态版。

第二阶段：资产与生成

- 做资产库。
- 做图片、视频、音乐、3D 生成面板。
- 所有工具调用结果写入任务记录和资产记录。

第三阶段：节点流

- 接入节点画布。
- 支持参考素材到生成工具的连接。
- 支持生成结果进入资产库。
- 支持人工确认节点。

第四阶段：构建与后期智能编排

- 做构建中心。
- 后期智能编排层读取当前项目、资产、节点、任务上下文。
- 后期智能编排快捷动作可触发 UI 任务。

第五阶段：双主题与验收

- 完整暗色主题。
- Playwright 截图验收。
- 修正溢出、重叠、空状态、动效问题。

## 12. 验收门槛

视觉验收：

- 第一屏必须明确显示 `TapTap Maker Plus` 和 `制造++`。
- 第一屏必须能看出当前项目与 MCP 状态。
- 第一屏必须有项目树、工作区、Inspector / 后期智能编排扩展位、任务队列的结构。
- 页面不能像临时 debug dashboard。
- 不使用黑色赛博风，不使用大面积霓虹，不使用泛紫色渐变。
- 不出现半亮半暗。

交互验收：

- 左栏可折叠。
- 右侧 Inspector / 后期智能编排扩展位可切换、可最小化、可调整宽度。
- 在 `1440x900` 桌面截图中，滚动中央内容后，顶栏、左栏、右栏、底部任务队列仍然可见。
- 切换模块后，当前项目、运行中的任务、右侧 Inspector 状态不被重置。
- 节点流 pan/zoom 时，外层工作台 shell 不移动、不消失。
- 资产库支持网格/列表。
- 生成任务进入任务队列。
- 错误可展开查看 raw result。
- 主题切换后布局不跳。

内容验收：

- 所有空状态都说明当前可做什么。
- 所有生成结果都能追溯来源工具和来源节点。
- 所有写入项目的动作都显示目标项目和目标路径。
- 构建、生成、写入项目、素材导入四类长任务都可追踪。

后续任何 UI 修改都应先对照本文档，再进入实现。若实现中发现本文档与真实 TapTap 页面证据或本地 MCP 能力冲突，先补证据，再更新文档，不直接凭印象改 UI。
